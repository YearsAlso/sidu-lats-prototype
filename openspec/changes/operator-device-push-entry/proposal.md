## Why

需求文档 V3.0(冻结)§4.3 M1-002 定义"检测记录提交时自动计算并追加哈希块"，§6.2 权限矩阵明确 OPERATOR 职能为"基础数据录入/自身范围内数据查看"，但当前 `prototype/operator/data-entry.html` 仅支持操作员手动上传 NTA 文件 + 基础表单录入，缺失以下能力:

1. **设备主动推送数据的接收与展示** — PRD M1-002 隐含设备端自动推送检测记录的能力(原型层面:mock 设备推送数据列表)
2. **推送数据的审核确认** — 操作员应能查看设备推送内容、逐条审核是否已录入、对未录入的推送数据补充信息后确认录入
3. **录入后的只读预览** — 已录入的推送数据可被选中查看详情

当前页面只有手动上传 + 最近录入表(mock 4 条全为"已录入")，无法演示"设备推送→操作员审核→补充信息→确认录入"的典型操作流程，削弱了 FDA 21 CFR Part 11 数据完整性(ALCOA+ 的 Attributable/Original)的演示价值。

## What Changes

- **修改** `prototype/operator/data-entry.html`，在现有手动录入表单**上方**新增「设备推送数据」卡片，含推送列表 + 预览面板:
  - **推送列表** (Table):展示 mock 设备推送数据(3 条:2 条待录入 + 1 条已录入)，字段含推送编号、来源设备、推送时间、检测类型、样品数量、数据摘要(文件名/大小)、提交状态(待录入/已录入)
  - **行选中交互**:点击推送列表行 → 展开预览面板，展示该条推送数据的详细信息(设备推送的原始数据 + 文件哈希等)
  - **待录入项**:预览面板底部展示补充录入表单(复用或内联现有表单字段:批次编号、检测类型(自动填入)、样品数量(自动填入)、检测设备(自动填入)、检测日期、负责人、备注)，填写后点「确认录入」→ 状态变为"已录入"，写入最近录入表
  - **已录入项**:预览面板仅展示只读详情(含录入人、录入时间、操作留痕摘要)，不可编辑
- **保留**现有的手动 NTA 上传 + 基础录入表单(独立卡片，不受推送列表影响)
- **保留**「我的最近录入」表格，推送数据确认录入后自动追加一行

### 不做什么

- **不**新建独立页面(在现有 data-entry.html 上扩展，遵守最小改动原则)
- **不**改其他角色页面、不改侧栏菜单结构、不改 operator dashboard
- **不**实现真实设备推送后端(纯 mock 数据)
- **不**引入外部审批流程(确认录入即终端状态，无 reviewer 审批链)
- **不**改 sessionStorage 鉴权流、不改跳转路径

## Capabilities

### New Capabilities
- `operator-device-push-list`: 设备推送数据列表 — mock 3 条设备推送数据，操作员可逐条查看、区分待录入/已录入状态
- `operator-push-preview`: 推送数据预览 — 选中推送行 → 展开详情面板，含设备原始数据预览 + 文件哈希
- `operator-push-entry`: 推送数据补充录入 — 待录入项的预览面板底部展示补充信息表单(自动填入已知字段)，点「确认录入」后状态变更 + 写审计留痕

### Modified Capabilities
- `operator-data-entry`: 增强 — 在原有手动上传 + 录入基础上，新增设备推送审核录入能力，两种录入方式(手动/推送)并存

## Impact

- **代码**:仅改 1 个文件 `prototype/operator/data-entry.html`(新增设备推送卡片 CSS + DOM + JS，约 150 行增量)
- **改动半径**:1 文件，约 150 行增量(远在最小改动阈值 3 文件/50 行增量边界内 — 50 行指纯代码行数而非增量；此处增量约 150 行，需用户确认)
- **依赖**:无新增依赖(纯 HTML/CSS/JS)
- **视觉**:antd 5 token(`#1677ff` 主色、`Table`/`Tag`/`Card` 组件还原、`#e6f4ff` 选中行底)
- **鉴权流**:不变(sessionStorage `lats_logged_in` + `lats_user` + `lats_role='operator'`)
- **跳转路径**:不变(侧栏仍 2 项:仪表盘 + 数据录入)
- **风险**:低风险(单文件增量，不改现有结构；推送列表在手动录入卡片上方，不冲突)；新增 JS 交互(行选中/预览面板切换/表单自动填入)需确保不破坏现有 `submitEntry` 逻辑
- **需求文档**:不改(已冻结)
