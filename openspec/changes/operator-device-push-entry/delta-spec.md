## 功能描述

`prototype/operator/data-entry.html` 新增设备推送数据审核录入能力。操作员可查看设备 mock 推送数据列表，选中待录入项预览详情、补充信息后确认录入；已录入项可选中查看只读摘要。

## 输入约束

- 推送数据列表:3 条 mock 数据(2 条待录入 + 1 条已录入)，页面加载时渲染
- 推送数据字段:推送编号、来源设备、推送时间、检测类型、样品数量、数据摘要(文件名+大小)、提交状态、文件哈希(展示态)
- 补充录入表单字段:批次编号(必填, text)、检测日期(必填, date, 默认今日)、备注(可选, textarea, max 500)
- 已录入数据额外字段:操作人、录入时间、批次编号、审计留痕哈希

## 输出约束

- 待录入行点击 → 预览面板展示推送数据详情(只读) + 补充录入表单
- 已录入行点击 → 预览面板展示推送数据详情(只读) + 录入摘要(只读)
- 确认录入 → 状态变为"已录入" → 行重新渲染 → 预览面板切换到录入摘要 → 最近录入表追加行
- 校验失败 → 错误提示可见(`#errorMsg` 展示, 符合 antd Feedback 原则)
- 关闭预览 → 预览面板隐藏, 选中状态清除

## 边界条件

- 切换选中行:同一时刻仅一行选中, 切换时预览面板内容更新
- 重复录入保护:已录入行( status==='done' )不展示补充录入表单
- 表单自动填入:选中待录入项时, 检测类型/样品数量/检测设备已在只读详情区展示(推送数据自带), 补充表单仅需填入批次编号+检测日期+备注
- 最近录入表追加:确认录入后, 在现有 4 条 mock 记录之前插入新行
- 不改变现有手动录入流程( NTA 上传 + 基础录入表单独立运作)

## 验收标准

1. 页面打开后,「设备推送数据」卡片在「NTA 原始数据上传」卡片上方, 展示 3 条推送数据(2 条 tag-warning"待录入" + 1 条 tag-success"已录入")
2. 点击待录入行 → 预览面板展开, 展示推送数据详情( 8 个只读字段) + 补充录入表单(批次编号/检测日期/备注 + "确认录入"按钮)
3. 检测日期自动填入今日; 批次编号留空 → 点击确认录入 → 错误提示"批次编号不能为空"
4. 填写批次编号 + 日期 → 点击确认录入 → 状态变为"已录入"( tag-success ), 预览面板切换到录入摘要(操作人/录入时间/批次编号/审计留痕哈希)
5. 最近录入表新增一行(批次编号/检测类型/样品数/检测日期/文件名/状态=已录入)
6. 点击已录入行 → 预览面板展示只读详情 + 录入摘要, 无补充录入表单
7. 点击预览面板关闭按钮 → 预览面板隐藏
8. 现有手动 NTA 上传 + 基础录入表单功能不受影响(上传/录入/重置正常)
